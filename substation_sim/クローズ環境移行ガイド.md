# クローズ環境移行ガイド

**開発VM でビルド → オフライン（クローズ系）環境へ移行する際の注意事項**

---

## 1. 動的リンクの確認

libIEC61850 のデフォルトビルドは**静的ライブラリ（`.a` / `.lib`）**を生成する。この場合、ライブラリコードは実行ファイルに埋め込まれるため、移行先に `.so` / `.dll` を持ち込む必要はない。ただし、ビルド設定によっては動的リンクになっている場合があるため、必ず確認すること。

### Linux での確認

```bash
# ビルドされたライブラリの形式を確認
ls -la libiec61850/build/src/
# libiec61850.a  ← 静的（OK）
# libiec61850.so ← 動的（要注意）

# ビルド済みバイナリの動的依存を確認
ldd substation_server
```

**OK（OS 標準ライブラリのみ）:**
```
linux-vdso.so.1
libpthread.so.0 => /lib/x86_64-linux-gnu/libpthread.so.0
libm.so.6       => /lib/x86_64-linux-gnu/libm.so.6
libc.so.6       => /lib/x86_64-linux-gnu/libc.so.6
```

**NG（libiec61850 が動的リンクされている → クローズ環境で起動不可）:**
```
libiec61850.so => not found
```

### Windows での確認

```powershell
# Visual Studio 付属の dumpbin を使用
dumpbin /dependents substation_server.exe
```

出力に `iec61850.dll` が含まれていなければ静的リンク済み。

## 2. リスク一覧と対策

| # | 問題 | 発生条件 | 影響 | 対策 |
|---|---|---|---|---|
| **①** | **glibc バージョン不一致** | 開発VM が新しい Linux、移行先が古い Linux | `GLIBC_2.xx not found` で起動不可 | ビルドは移行先と同じ（または古い）OS で行う。または完全静的リンク |
| **②** | **MSVC ランタイム不在** | Visual Studio でビルドした場合 | `vcruntime140.dll` 等が見つからず起動不可 | MinGW を使うか、MSVC ランタイム再頒布パッケージを同梱 |
| **③** | **MinGW ランタイム不在** | MinGW でビルドし `-static` を付けなかった場合 | `libgcc_s_dw2-1.dll`、`libwinpthread-1.dll` が見つからず起動不可 | `-static` フラグで静的リンク |
| **④** | **Windows XP 互換性** | 新しい Windows SDK / MSVC でビルドした場合 | XP で実行不可 | XP 互換のツールチェイン（VS2015 以前、または MinGW）を使用 |
| **⑤** | **pip が使えない** | クローズ環境にインターネット接続なし | `pip install` 不可 | 事前に wheel をダウンロードして持ち込む |
| **⑥** | **Windows Update 不可** | クローズ環境 | DCOM Hardening のパッチ状態がずれる | 開発VM でパッチ状態を合わせておく |
| **⑦** | **時刻同期不可** | NTP なし | IEC 61850 タイムスタンプ、OPC DA 品質に影響 | 手動同期 or VirtualBox ホスト時刻同期機能 |
| **⑧** | **ライセンス認証** | Windows 評価版 / KEPServerEX | 猶予期間切れで使用不可 | オフライン電話認証。KEPServerEX のデモモードはオフラインで動作 |

## 3. 完全静的リンクでのビルド方法

### Linux

```bash
cmake .. -DLIBIEC61850_HOME=../../libiec61850 \
         -DCMAKE_EXE_LINKER_FLAGS="-static"
make
```

確認:
```bash
ldd substation_server
# → "not a dynamic executable" と出れば完全静的

file substation_server
# → "statically linked" と表示される
```

> **注:** glibc の完全静的リンクは NSS 関連（`getaddrinfo` 等）で警告が出る場合があるが、本シミュレータの用途では実害なし。

### Windows (MinGW)

```bash
cmake .. -DLIBIEC61850_HOME=../../libiec61850 \
         -G "MinGW Makefiles" \
         -DCMAKE_EXE_LINKER_FLAGS="-static -static-libgcc -static-libstdc++"
mingw32-make
```

### Windows (Visual Studio)

CMakeLists.txt に以下を追加し、CRT を静的リンクに切り替える:

```cmake
if(MSVC)
    set(CMAKE_C_FLAGS_RELEASE "${CMAKE_C_FLAGS_RELEASE} /MT")
    set(CMAKE_C_FLAGS_DEBUG "${CMAKE_C_FLAGS_DEBUG} /MTd")
endif()
```

## 4. Windows XP をターゲットにする場合の互換性

VM2（RTU/ゲートウェイ）を Windows XP で構成する場合、以下のバージョン制約がある:

| ソフトウェア | XP 対応バージョン | 備考 |
|---|---|---|
| **libIEC61850** | 最新版で対応 | MinGW または VS2015 以前でビルド |
| **Python** | **3.4.4** が XP 最終対応版 | 3.5 以降は XP 非サポート |
| **OpenOPC** | Python 2.7 / 3.4 向け | pywin32 も XP 対応版を使用 |
| **pyodbc** | 4.0.x | XP 対応の最終系列 |
| **MatrikonOPC** | XP 対応版あり | ダウンロード時に OS を指定 |
| **KEPServerEX** | **5.x** が XP 対応 | 6.x は Windows 7 以降 |
| **SQL Server Express** | **2008 R2** が XP 最終対応 | 2012 以降は非サポート |
| **Wireshark** | **1.12.x** が XP 最終対応 | WinPcap を使用（Npcap は XP 非対応） |

## 6. Python パッケージの事前ダウンロード

クローズ環境では `pip install` が使えないため、開発VM（インターネット接続あり）で事前にパッケージをダウンロードしておく。

### ダウンロード手順（開発VM で実行）

```bash
# 指定ディレクトリに wheel / tar.gz をダウンロード
pip download openopc-da -d ./python_packages/
pip download pyodbc -d ./python_packages/
pip download pywin32 -d ./python_packages/

# XP (Python 3.4) 向けの場合はバージョンとプラットフォームを指定
pip download openopc-da --python-version 3.4 --only-binary=:all: \
    --platform win32 -d ./python_packages/
```

### クローズ環境でのインストール手順

```bash
# オフラインインストール（pip の --no-index オプション）
pip install --no-index --find-links=./python_packages/ openopc-da
pip install --no-index --find-links=./python_packages/ pyodbc
```

> **pywin32 は `.exe` 形式のインストーラが配布されている場合があり、その場合はダブルクリックでインストール可能。**

## 7. 移行前の最終チェックリスト

### ビルド成果物

- [ ] `ldd`（Linux）/ `dumpbin`（Windows）でバイナリの動的依存を確認
- [ ] `libiec61850.so` / `iec61850.dll` への依存がないことを確認
- [ ] MSVC の場合 `vcruntime140.dll` 依存がないか、同梱したか確認
- [ ] MinGW の場合 `-static` フラグでビルドしたか確認

### ソフトウェアパッケージ

- [ ] 全てのインストーラ / ISO がオフラインメディアにコピー済み
- [ ] Python パッケージが wheel / tar.gz 形式でダウンロード済み
- [ ] KEPServerEX / MatrikonOPC のインストーラ入手済み
- [ ] OPC Core Components のインストーラ入手済み
- [ ] SQL Server Express の**オフラインインストーラ**（Web installer ではなく）入手済み

### OS 互換性

- [ ] VM2 が Windows XP の場合、XP 対応バージョンのソフトを選定済み
- [ ] 開発VM と移行先 VM の OS / glibc バージョンの差異を確認済み

### ネットワーク

- [ ] 各 VM の固定 IP アドレス設計が決定済み
- [ ] hosts ファイルの内容を事前に準備済み
- [ ] DCOM 設定値（第9章の各シナリオ）を手順書として準備済み

### ライセンス

- [ ] Windows 評価版の猶予期間を確認（XP: なし、7: 30日、10: 90日）
- [ ] KEPServerEX デモモードがオフライン動作することを確認済み
- [ ] SQL Server Express のライセンス条件を確認済み（オフライン利用可）
